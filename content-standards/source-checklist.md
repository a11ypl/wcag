# Source checklist

Checklista źródeł W3C, technik i failures.

## Źródła obowiązkowe

- [ ] WCAG 2.2 dla danego kryterium.
- [ ] Understanding WCAG dla danego kryterium.
- [ ] How to Meet / Quick Reference dla danego kryterium.

## Techniki W3C

- [ ] Każda technika została sprawdzona w How to Meet / Quick Reference dla danego kryterium.
- [ ] Technika pasuje do przykładu i zakresu tutoriala.
- [ ] Technika nie została dopisana z pamięci.
- [ ] Nazwa techniki i link prowadzą do stabilnego adresu W3C.
- [ ] Jeśli technika jest tylko kontekstowa, a nie wystarczająca dla danego kryterium, nie opisuj jej jako wystarczającej.

## Failures

- [ ] Failure został sprawdzony w How to Meet / Quick Reference albo Understanding dla danego kryterium.
- [ ] Failure jest istotny dla treści tutoriala.
- [ ] Jeśli Quick Reference nie wskazuje osobnej listy failures, zapisz to w source QA.
- [ ] Informacji o braku failures nie dodawaj do publicznej sekcji `Źródła`.

## Source QA

- [ ] Dopisz status źródeł w `wcag-prostym-jezykiem-source-qa.md`.
- [ ] Wpisz datę checkpointu.
- [ ] Wypisz zweryfikowane źródła podstawowe.
- [ ] Wypisz zweryfikowane techniki.
- [ ] Wypisz zweryfikowane failures albo notatkę, że ich nie wskazano.
- [ ] Dodaj uwagi zakresowe, jeśli kryterium łatwo pomylić z innym.
