# Checklista przed wdrożeniem A11yFirst Lab

## Treść

- [ ] Każde pytanie ma dokładnie jedną poprawną odpowiedź.
- [ ] Każda odpowiedź ABCD ma własny feedback.
- [ ] Prawda/Fałsz ma feedback dla obu wyborów.
- [ ] Feedback wyjaśnia, dlaczego odpowiedź jest dobra albo błędna.
- [ ] Kryteria WCAG są przypisane poprawnie.
- [ ] Rekomendacje naprawcze są praktyczne.

## Dostępność

- [ ] Wszystko działa klawiaturą.
- [ ] Fokus jest widoczny.
- [ ] Nie ma pułapki klawiaturowej.
- [ ] Skiplink działa.
- [ ] Pytania ABCD są radiobuttonami.
- [ ] Feedback jest dostępny dla czytnika ekranu.
- [ ] Kontrast tekstu spełnia minimum 4.5:1.
- [ ] Fokus i kontrolki spełniają minimum 3:1.
- [ ] Nie ma informacji tylko kolorem.
- [ ] Aplikacja działa przy 200% i 400% zoom.
- [ ] Aplikacja działa przy szerokości 320 CSS px.
- [ ] Działa `prefers-reduced-motion`.
- [ ] Działa w forced colors / wysokim kontraście.

## Postęp

- [ ] Postęp zapisuje się po odpowiedzi.
- [ ] Postęp zostaje po odświeżeniu strony.
- [ ] „Kontynuuj” działa.
- [ ] „Powtórz błędne odpowiedzi” działa.
- [ ] Eksport działa.
- [ ] Import poprawnego pliku działa.
- [ ] Import błędnego pliku pokazuje konkretny błąd.
- [ ] Reset wymaga potwierdzenia.

## Techniczne

- [ ] `npm run build` działa.
- [ ] `npm run test` działa.
- [ ] Testy axe nie pokazują critical/serious.
- [ ] Testy Playwright klawiatury przechodzą.
- [ ] README zawiera deploy.
- [ ] W kodzie nie ma haseł ani sekretów.
